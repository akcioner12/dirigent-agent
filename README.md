# Dirigent Agent

Оркестратор + субагенты для автоматизации SMM в нише крипто/AI/форекс.
Фаза 1 (MVP): мониторинг Telegram-каналов конкурентов → переделка через Claude →
одобрение в Telegram → публикация в целевой канал.

## Архитектура

- `main.py` — точка входа, Telegram-бот, расписание сканирования.
- `orchestrator.py` — связывает мониторинг → rewrite → approval → publish.
- `subagents/tg_monitor.py` — читает каналы-источники через Telethon (user-аккаунт).
- `subagents/rewriter.py` — переделка текста через Claude API.
- `db.py` — SQLite, очередь постов со статусами pending/approved/rejected/published.
- `config.py` — список источников и настройки.

## Важно: два разных Telegram-доступа

1. **BOT_TOKEN** (от @BotFather) — публикует одобренные посты в твой канал и
   присылает тебе уведомления с кнопками.
2. **TG_API_ID / TG_API_HASH** (с my.telegram.org) — нужен для **чтения** чужих
   публичных каналов. Обычный бот не может читать каналы, где он не админ —
   поэтому мониторинг конкурентов идёт через Telethon под твоим личным аккаунтом.

## Переменные окружения (Railway → Variables)

См. `.env.example`. Локально — скопируй в `.env` и заполни.

## Первый запуск

При первом запуске Telethon попросит код подтверждения из Telegram —
это нужно будет ввести один раз в логах Railway (либо локально перед деплоем,
чтобы сгенерировать `monitor_session.session` и закоммитить его как Volume/Variable).

## Команды бота

- `/scan` — запустить цикл мониторинга вручную
- `/status` — сколько постов опубликовано сегодня
